2.编写配置文件覆盖 entry/output

webpack.config.js 是 webpack 默认配置的文件名，在根目录下创建：

path.resolve() 方法会把一个路径或者是路径片段的序列解析为一个绝对路径
__dirname：当前模板的文件夹名称。

执行 npm run build 打包 js 文件
会发现生成了 dist 文件，并且生成了两个打包后的文件

这跟 AMD 的引入方式有关，如果在 app.js 中注释掉 AMD 的写法，则只会打包出一个 bundle.js 文件

在实际写代码的时候，最好使用 ES6 和 CommonJS 的规范来写

当你注释 AMD 后，打包后的 dist 中有多个文件，这是因为打包的时候没有先删除掉 dist 文件，在打包，我们需要使用一个插件来帮助我们实现：github 链接：clear-webpack-plugin
1.安装插件
npm install clean-webpack-plugin –save-dev
